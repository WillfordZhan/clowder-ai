# PPT Low-Fi Authoring — 低保真 MD 制作规范

> ppt-forge 的核心 ref。
> 来源：2026-05-28 LLE 架构图实战 + 2026-05-29 试用期总结实战。
> 两次验证的成熟模式：架构猫写低保真 MD → imagegen 猫 imagegen 逐页出精美图。

## 核心原则

**低保真 MD 是"给 AI 看的蓝图"——不是给人看的文档。**

- 每页一个 section，用 ASCII art 画清布局和信息分区
- 视觉指引用文字精确描述配色、字体、图表风格
- 内容全量保留，用结构化图表承载密度
- 写完后交给imagegen 猫（或其他有 imagegen 能力的猫）逐页生成精美图

## 适用场景

| 场景 | 适合 |
|------|------|
| 做 PPT / 演示文稿 / 幻灯片 | 适合 |
| 帮朋友/外部做 PPT | 适合 |
| 架构图 / 技术蓝图 | 适合 |
| 快速出高质量图 | 适合 |
| 工作汇报 / 转正答辩 / 客户提案 | 适合 |

## 分页规划方法

### 第 1 步：内容分析

阅读全部原始内容，识别：
- **核心板块**：几个大的独立主题
- **数据点**：有冲击力的数字（800+、50+、165 等）
- **流程/关系**：可以图表化的逻辑链
- **层级**：总→分的信息结构

### 第 2 步：页面类型分配

| 页面类型 | 用途 | 典型视觉形态 |
|---------|------|-------------|
| 封面 | 标题+基本信息 | 品牌色带+居中标题 |
| 总览/全景 | 一页看全貌 | 矩阵卡片/思维导图/N 宫格 |
| 详情页 | 展开一个板块 | 双栏/流程图/表格+数据 |
| 方法论 | 提炼核心方法 | 横向卡片/公式/流程 |
| 路线图 | 时间维度规划 | 时间轴/里程碑/甘特 |
| 对比页 | 双栏对比 | 左右分栏/表格/雷达图 |
| 总结页 | 收获/不足/致谢 | 双栏+大字致谢 |

### 第 3 步：页面数量控制

- **华为/document 风格**：每页密排，6-12 页常见
- **Apple/presentation 风格**：每页一个点，15-25 页常见
- **内容全保留时**：合并关联板块到同一页（用分栏/标签/卡片承载）
- **内容可取舍时**：先砍再排

## ASCII Art 规范

### 字符集

```
边框：┌ ┐ └ ┘ │ ─ ═ ║ ╔ ╗ ╚ ╝ ╠ ╣ ╬ ╤ ╧
箭头：→ ← ↑ ↓ ▶ ◀ ▲ ▼ ──▶ ◀──
分隔：╠═══╣  ╠═══╤═══╣（竖线分栏）
强调：══（双线标题框）
```

### 布局原则

- **大小反映比例**：占页面 1/3 的区域，ASCII art 也占 1/3 宽度
- **层级用缩进**：顶级元素贴左，子元素缩进 2-4 格
- **卡片用方框**：每个独立信息块用 `┌──┐` `└──┘` 围起
- **流程用箭头**：`──▶` 或 `│ ▼`
- **分栏用竖线**：`╤` `│` `╧`

### 好的 ASCII art 示例

```
┌══════════════════════════════════════════════┐
║  页面标题                                     ║
╠═══════════════════════╤══════════════════════╣
║  【左栏标题】          │  【右栏标题】         ║
║                       │                      ║
║  ┌─────────────┐     │  ┌────────────────┐  ║
║  │ 卡片 1      │     │  │ 表格           │  ║
║  │ · 要点 A    │     │  │ ┌───┬──────┐  │  ║
║  │ · 要点 B    │     │  │ │ A │ 说明 │  │  ║
║  └─────────────┘     │  │ ├───┼──────┤  │  ║
║        │              │  │ │ B │ 说明 │  │  ║
║        ▼              │  │ └───┴──────┘  │  ║
║  ┌─────────────┐     │  └────────────────┘  ║
║  │ 卡片 2      │     │                      ║
║  └─────────────┘     │  ┌───────────────┐   ║
║                       │  │ 大数据高亮     │   ║
║                       │  │   800+        │   ║
║                       │  └───────────────┘   ║
╚═══════════════════════╧══════════════════════╝
```

## 视觉指引格式

每页 ASCII art 下方附 `**视觉指引**：` 段落。视觉指引**不重新发明配色**——直接引用风格 preset 的参数名。

### 与风格 preset 的联动

低保真稿选定风格 preset 后（如 `ppt-style-huawei.md`），视觉指引里直接用 preset 的术语：

| preset 定义的参数 | 视觉指引里这样写 |
|-----------------|-----------------|
| 主色红 `#D33941` | "品牌红" |
| 深灰 `#555757` | "深灰正文" |
| 极浅灰 `#DDDDDD` | "极浅灰边框" |
| 28-36pt 加粗 Arial | "KPI 大数字" |
| 6-9pt 灰色斜体 | "注释区格式" |
| 页面结构"KPI 数据看板" | "参照 preset §七 KPI 数据看板" |

**不需要在每页视觉指引里重复写色值**——preset 是单一真相源。

### 视觉指引模板

```markdown
**视觉指引**：
- 页面类型：{引用 preset §七 的模式名，如"并列卡片页"}
- 布局：{双栏/三栏/矩阵/流程/...}
- 标题：{品牌红 / 黑色，对应 preset §五 字号层级}
- 卡片/区域：{浅灰底 + 极浅灰边框 / 白底 / ...}
- 关键数据：{哪些数字用 KPI 大数字格式（品牌红 28-36pt）}
- 流程/箭头：{品牌红 / 浅灰}
- 底部总结条：{品牌红底白字 / 无}
- 装饰元素：{品牌飘带 / 科技光线 / 几何点阵 / 渐变色带 / 无（参照 preset §装饰元素层）}
- 注释区：{数据来源，注释区格式}
```

**要点**：
- 只描述风格意图，不描述 CSS——这是给 imagegen 的指令
- 配色用 preset 术语（"品牌红""深灰""极浅灰"），不重复色值
- 每条指引都是 imagegen 可执行的
- 没把握的页面类型，直接写"参照 preset §七 {模式名}"

### 面板内容类型标注

一页 PPT 通常由多个面板（信息块）组成。每个面板在 ASCII art 里画出位置后，用**内容类型标签**告诉 imagegen 猫这个面板放什么：

| 标签 | 含义 | imagegen 怎么处理 |
|------|------|-------------------|
| `[文字]` | 纯文字要点/描述 | 按 preset 字号排版 |
| `[数据]` | KPI 数字/指标 | 品牌红大号 + 灰色单位 |
| `[图表]` | 柱状图/饼图/折线图 | 按 preset §八 图表配色 |
| `[表格]` | 对比表/参数表 | 灰色表头 + 红色高亮 |
| `[流程]` | 箭头流程/步骤 | 品牌红编号 + 箭头 |
| `[实拍]` | 真实产品照片（客户提供） | 标注"此处放客户提供的照片：{描述}" |
| **`[概念图]`** | **AI 生成的概念渲染图** | **按描述生成，详见下方格式** |
| `[公式]` | 数学/工程公式 | 黑色正文，变量用斜体 |

### 概念渲染图描述格式

当一个面板需要 AI 生成概念渲染图时（不是真实照片，而是"看起来像照片的概念图"），用以下格式描述：

```markdown
┌──────────────────────────┐
│ [概念图]                  │
│ 主体：{描述画什么}         │
│ 风格：{科技感/写实/等距/...}│
│ 氛围：{未来感/专业/温暖/...}│
│ 参考：{如有参考图片或关键词} │
└──────────────────────────┘
```

**示例**（参考华为 ISCAS PPT）：

```markdown
┌──────────────────────────┐
│ [概念图]                  │
│ 主体：赛博朋克风格的猫咖   │
│       室内，霓虹灯 + 猫猫  │
│ 风格：3D 渲染，科技感      │
│ 氛围：未来感 + 温暖        │
└──────────────────────────┘

┌──────────────────────────┐
│ [概念图]                  │
│ 主体：服务器机房全景       │
│       蓝色 LED + 整齐排列  │
│ 风格：写实渲染             │
│ 氛围：专业、大气           │
└──────────────────────────┘
```

**关键**：概念图描述要**具体到画面内容**，不能只写"搞一张好看的图"。imagegen 猫根据描述 + preset 配色（品牌红点缀）生成。

## 精美图生成指引（末尾总章）

低保真 MD 末尾必须有一个"精美图生成指引"section。**核心是指向风格 preset**，不在这里重复写完整配色——preset 是单一真相源。

### 总章必填项

| 项目 | 怎么写 |
|------|--------|
| **风格 preset 引用** | `遵循 ppt-style-huawei.md`（imagegen 猫据此加载完整配色/字体/布局规范） |
| 品牌主色覆盖（如需） | 如朋友公司不用华为红，这里写覆盖色值 |
| 排版规格 | 16:9 / 4:3，横版/竖版 |
| 密度原则 | 如"极致密排，每页至少一个图表化元素" |
| **逐页生成清单** | P1-PN 每页标题 + 主要图表元素 |

### 总章示例

```markdown
## 精美图生成指引（给 imagegen 猫）

**风格 preset**：遵循 `ppt-style-huawei.md`（华为式专业汇报风格）
- 如需覆盖品牌主色：{色值}（默认 #D33941）

**排版**：16:9 横版，每页独立一张精美图

**密度**：极致密排，每页至少一个图表化元素

**需要生成的图**：
| 序号 | 页面 | 页面类型（preset §七） | 主要图表元素 |
|---|---|---|---|
| P1 | 封面 | — | 品牌红色带 + 标题居中 |
| P2 | 工作总览 | 并列卡片页 | 五板块矩阵卡片 + 总结条 |
| P3 | 代码检视 | 图表分析页 | 双栏：闭环流程 + KPI 数据 |
| ... | ... | ... | ... |
```

## 与 imagegen 猫的交接协议

低保真 MD 写完后，交给有 imagegen 能力的猫（imagegen 猫 / 云端 Codex）：

1. **发低保真 MD 文件路径**（不贴全文，太长）
2. **说明生成方式**：逐页生成，每页一张独立图片
3. **说明输出位置**：同目录 `assets/` 子目录
4. **关键约束**：
   - 用原生 imagegen 直出 raster PNG，不用 SVG/HTML 手工合成
   - 每页图片命名：`p{N}-{简短描述}.png`（如 `p2-overview.png`）
   - 严格遵循配色方案，不自由发挥
   - 文字内容以低保真 MD 为准，不删不加

## Common Mistakes

| 错误 | 后果 | 修复 |
|------|------|------|
| ASCII art 太简陋（只有标题列表） | imagegen 猜布局，结果不对 | 画清每个区域的位置和大小 |
| 视觉指引缺配色 | imagegen 用默认配色 | 每页都引用风格 preset |
| 没写数据高亮方式 | 关键数字淹没在正文中 | 标注哪些数字要大号/高亮 |
| 直接贴大段原文 | imagegen 不知道怎么排版 | 先做信息设计（分卡片/分层级/转图表） |
| 没写精美图生成指引 | imagegen 猫要反复问风格 | 末尾必有总章 |
| 把视觉指引写成 CSS 代码 | imagegen 不吃 CSS | 用自然语言描述视觉意图 |
